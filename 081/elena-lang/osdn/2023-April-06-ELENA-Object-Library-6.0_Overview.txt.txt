= お断り

elena-lang.github.io/api/ 以下の翻訳物です。

あまり大したことは書かれていないので翻訳しないかもしれません。必要になれば、掲載記事をいつものところに移動してから作業するか別の作業部屋を用意して考えます。というより、ELENA や Objeck はランゲージリファレンスとライブラリリファレンス、チュートリアルのドキュメントのファイルフォーマットを統一して集中管理していないので扱いづらいですね（検索できぬ）。なんでマイナー言語は揃いも揃ってドキュメントシステムが時代に合わない打製石器ばっかりなんだか。

せめて Gravity 言語くらいの水準にしたいところ(ELENA は ghpage の機能を全く引き出せていない)。あと、 FFI まわりの資料が文書化されていないのでやる気が出ないのよね。

== 読んでおいたほうがいいところ

読む前に httpd サーバがないと正常にブラウザで読めません。なんでローカルで読む想定をしていないんだか。いちおう、ldoc とかいう自前ツールでドキュメントを生成しているらしいのですが、UTF-8 非対応らしい上に、出力結果は HTML 3.x の水準で止まっている残念なツール(改造しないと Markdown すら出力できない)なので、素直にランタイムライブラリのソースコードを読んだほうがいいと思います (非公開機能が多すぎて割と読みづらい)。

=== 翻訳作業方法

 * src50, src60 の *.l がELENA 側のライブラリソースコードです。なんか検索すると非公開機能が色々合って混乱すると思いますが気にしないでください。

 * /// がドキュメントコメントなので、それを grep して釜に投げ込めばいいだけです。

いろいろと至らなすぎて辞めたくなってきた。
